## 🔍 详细规则解读

### 一、中文队列规则

#### 1. 入池规则（两种触发条件）
**条件A：单次严重扣罚**
- **触发条件**：出现"当月为零"或以上程度质检扣罚
- **入池时间**：以日报时间为准，当日入池
- **关键点**：单次严重错误即可入池

**条件B：连续轻微扣罚**
- **触发条件**：连续双周出现"当通为零"次数达到2次
- **入池时间**：以第二次出现"当通为零"的日报时间为准，当日入池
- **关键点**：连续两周内累计2次轻微错误
- **"连续双周"定义**：看的是自然周，不是任意14天

#### 2. 出池规则
- **基本规则**：连续14天未出现"当通为零"及以上程度质检扣罚（包含入池当天）
- **特殊规则**：若在14天内再次出现"当通为零"及以上扣罚，则入池时间刷新
- **刷新机制**：重新开始计算14天周期
- **关键点**：必须连续14天无任何扣罚（含入池当天）

#### 3. 时间计算示例
```
示例1：单次严重扣罚入池
- 3月28日：出现"当月为零"扣罚
- 3月30日：日报发布（反映3月28日工单）
- 入池时间：3月30日（日报时间）
- 最早出池时间：4月13日（3月30日+14天）

示例2：连续轻微扣罚入池
- 第1周：3月14日出现"当通为零"
- 第2周：3月21日出现"当通为零"（第二次）
- 3月23日：日报发布（反映3月21日工单）
- 入池时间：3月23日（第二次的日报时间）
- 最早出池时间：4月6日（3月23日+14天）

### 3. "连续双周"详细解释
**业务规则澄清**（2026-03-30 10:06 成哥确认）：
- **"连续双周"定义**：看的是自然周，不是任意14天
- **出池观察周期**：按自然日连续计算，包含入池当天
- **示例说明**：
  - A今天3月30号入池
  - 观察周期就是3月30号-4月12号（共14天，含入池当天）
  - 这14天是连续的自然日，不考虑工作日或节假日

**自然周 vs 任意14天区别**：
- **自然周**：按周一到周日计算，跨周才算"连续双周"
- **任意14天**：任意连续14天都算
- **当前规则**：采用自然周概念，但出池计算按自然日连续

**自动化处理要点**：
1. 入池判断时："连续双周"按自然周判断
2. 出池计算时：按自然日连续计算（含入池当天）
3. 节假日处理：自然日连续计算，不考虑节假日
```

### 二、英文队列规则

#### 1. 入池规则
- **触发条件**：出现"当通为零"及以上程度质检扣罚
- **入池时间**：以日报发布时间为准
- **关键点**：当天日报发布中出现"当通为零"的伙伴，当日入池

#### 2. 出池规则
- **基本规则**：连续7天个人未出现"当通为零"及以上程度质检扣罚
- **特殊规则**：若在连续7天中再次出现扣罚，则入池时间刷新
- **刷新机制**：重新开始计算7天周期
- **关键点**：必须连续7天无任何扣罚

#### 3. 时间计算示例
```
示例：
- 3月28日：出现"当通为零"扣罚
- 3月30日：日报发布（反映3月28日工单）
- 入池时间：3月30日（日报发布时间）
- 最早出池时间：4月6日（3月30日+7天）
```

### 2. 入池判断逻辑
```python
def check_entry_conditions(staff_id, team_type, penalty_history):
    """
    判断是否需要入池
    team_type: "chinese" 或 "english"
    penalty_history: 最近N天的扣罚记录
    """
    
    if team_type == "chinese":
        # 中文队列规则
        # 条件A：单次严重扣罚
        if has_severe_penalty_recently(penalty_history):
            return True, "severe_penalty"
        
        # 条件B：连续双周2次轻微扣罚（按自然周判断）
        if count_minor_penalties_in_consecutive_natural_weeks(penalty_history, weeks=2) >= 2:
            return True, "continuous_minor"
    
    elif team_type == "english":
        # 英文队列规则
        # 条件：单次轻微及以上扣罚
        if has_any_penalty_recently(penalty_history):
            return True, "any_penalty"
    
    return False, None

def count_minor_penalties_in_consecutive_natural_weeks(penalty_history, weeks=2):
    """
    判断连续自然周内的轻微扣罚次数
    规则：连续双周（自然周）内出现2次"当通为零"
    
    参数：
    - penalty_history: 扣罚记录列表，每个元素包含(date, type)
    - weeks: 连续周数，默认为2（双周）
    
    返回：
    - 连续自然周内的轻微扣罚次数
    """
    # 获取当前日期
    current_date = datetime.now().date()
    
    # 计算过去weeks个自然周
    minor_penalties_count = 0
    minor_penalties_dates = []
    
    # 收集过去weeks*7天内的轻微扣罚
    for penalty in penalty_history:
        penalty_date, penalty_type = penalty
        days_diff = (current_date - penalty_date).days
        
        # 只考虑过去weeks*7天内的扣罚
        if 0 <= days_diff <= weeks * 7:
            if penalty_type == "当通为零":
                minor_penalties_count += 1
                minor_penalties_dates.append(penalty_date)
    
    # 检查是否分布在连续的自然周内
    if minor_penalties_count >= 2:
        # 按周分组
        weekly_groups = {}
        for date in minor_penalties_dates:
            week_number = date.isocalendar()[1]  # ISO周数
            if week_number not in weekly_groups:
                weekly_groups[week_number] = []
            weekly_groups[week_number].append(date)
        
        # 检查是否有连续的自然周都有扣罚
        week_numbers = sorted(weekly_groups.keys())
        for i in range(len(week_numbers) - 1):
            if week_numbers[i+1] - week_numbers[i] == 1:
                # 找到连续自然周
                total_in_consecutive_weeks = sum(
                    len(weekly_groups[week_numbers[i]]), 
                    len(weekly_groups[week_numbers[i+1]])
                )
                if total_in_consecutive_weeks >= 2:
                    return total_in_consecutive_weeks
    
    return 0
```

### 3. 出池判断逻辑
```python
def check_exit_conditions(staff_id, team_type, entry_date, penalty_history_since_entry):
    """
    判断是否可以出池
    entry_date: 入池日期
    penalty_history_since_entry: 入池后的扣罚记录
    """
    
    if team_type == "chinese":
        required_days = 14
    elif team_type == "english":
        required_days = 7
    else:
        return False
    
    # 检查入池后是否有扣罚
    if has_penalty_since_entry(penalty_history_since_entry):
        return False  # 有扣罚，不能出池
    
    # 计算连续无扣罚天数（含入池当天）
    consecutive_days = calculate_consecutive_days_without_penalty(
        entry_date, penalty_history_since_entry
    )
    
    return consecutive_days >= required_days
```

### 4. 时间刷新逻辑
```python
def refresh_entry_date(staff_id, new_penalty_date, current_entry_date):
    """
    特殊规则：出现新扣罚时刷新入池时间
    """
    # 如果新扣罚日期晚于当前入池日期，则刷新
    if new_penalty_date > current_entry_date:
        return new_penalty_date
    else:
        return current_entry_date
```

## 📅 时间概念应用

### 关键时间点
1. **工单产生日**：实际发生扣罚的日期
2. **质检日报文件发出日**：日报发布日期（=自然日）
3. **入池执行日**：质检日报文件发出日
4. **出池计算起点**：入池执行日（含当天）

### 时间延迟考虑
```
实际业务时间线：
工单产生日（扣罚发生） → 工单抽检日 → 质检日报文件发出日（入池执行）

示例：
- 3月28日：工单产生，发生扣罚
- 3月29日：工单抽检
- 3月30日：日报发布，执行入池
- 入池日期记录为：3月30日
```

### 节假日处理
- 如果日报延迟发布，入池日期仍按日报发布时间
- 出池计算按自然日连续计算，不考虑工作日
- 长假期合并日报需要特殊处理

## 📋 每日/每周任务要求

### 中文队列任务

#### 每日需做（针对流程及以上程度扣罚）
1. **扣罚分析**：日报出具后第一个工作日完成，提交组长审核
2. **规则抄写**：
   - 普通流程：抄写10遍
   - 重大及以上：抄写20遍
   - 第一个工作日内完成，发送到质检入池群通晒
3. **数据统计**：完成个人日度、周度、月度质检数据统计
4. **内容讲解**：每天向训练营其他伙伴进行内容讲解（飞书视频会议）
5. **完成时间**：每日16:00前完成
6. **开始时间**：入池后第一个工作日开始

#### 每周需做
1. 参加质检考核
2. 每周质检问题分析+考试内容复盘分析
3. 每周1-2次质检规则文档学习
4. 质检抄写

### 英文队列任务

#### 每日需做（针对未满分工单扣罚）
1. **扣罚分析**：日报出具后第一个工作日完成，包含：
   - 扣罚点
   - 扣罚原因
   - 扣罚原因分析
   - 后续解决方案
2. **提交审核**：提交组长审核
3. **完成时间**：每日16:00前完成
4. **开始时间**：入池后第一个工作日开始

#### 每周需做
1. 参加质检考核

## ⚠️ 特殊规则和注意事项

### 1. 入池时间刷新规则
- **中文队列**：14天内再次出现扣罚，入池时间刷新
- **英文队列**：7天内再次出现扣罚，入池时间刷新
- **刷新效果**：重新开始计算出池周期

### 2. "包含入池当天"规则
- 出池计算时，入池当天计入连续天数
- 示例：3月30日入池，最早出池日：
  - 中文：4月13日（3月30日+13天=4月12日，含当天为14天）
  - 英文：4月6日（3月30日+6天=4月5日，含当天为7天）

### 3. 扣罚程度定义
- **当通为零**：单通对话为零分（轻微）
- **当月为零**：整月累计为零分（严重）
- **其他严重**：文档中未明确定义的其他严重扣罚

### 4. 任务调整机制
- "每日需做"和"每周需做"内容会阶段性调整
- 中文队列：关注"质检训练营群"公告
- 英文队列：关注"质检重大错误预防分析处理组"公告

## 🔄 与时间概念的关联

### 四个时间概念的应用
1. **自然日**：出入池操作的执行日
2. **质检日报文件发出日**：入池判断的依据日
3. **工单抽检日**：扣罚识别的中间环节
4. **工单产生日**：扣罚实际发生的日期

### 时间链条验证
```
自动化系统需要验证：
1. 日报文件名中的日期 = 工单产生日
2. 工单抽检日 = 工单产生日 + 1天
3. 质检日报文件发出日 = 工单抽检日 + 1天
4. 入池执行日 = 质检日报文件发出日
```
