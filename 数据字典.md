## 📊 扣罚类型体系（基于《International技术支持质检规则2.0》）

基于成功读取的质检评分规则表，完整的扣罚类型体系如下：

### 1. 按扣罚程度分类（五级体系）

#### 🔴 **第一级：团队红线（一票否决）**
- **性质**：最严重，直接开除
- **包含**：
  1. 数据造假
  2. 泄露公司机密
  3. 职场内打架斗殴
  4. 辱骂客户、同事

#### 🔴 **第二级：虚假工单（诚信问题）**
- **性质**：严重诚信问题
- **首次**：当月绩效0.5
- **再犯**：当月绩效为零或劝退
- **年度累计**：累计计算

#### 🔴 **第三级：当月为零（严重扣罚）**
- **性质**：严重业务错误
- **触发入池**：✅ 单次即触发入池（中文队列）
- **包含**：
  1. 系统升级无风险提示
  2. 远程连接未征得同意（三点必须全部达标）
  3. 违规操作导致严重后果
  4. 无故挂断客户电话
  5. 造成公司或客户损失
  6. 挑衅、讽刺、辱骂客户
  7. 泄露公司机密
  8. 敷衍了事冷暴力（抢话3次及以上）

#### 🟡 **第四级：当通为零（轻微扣罚）**
- **性质**：轻微业务错误
- **触发入池**：
  - 中文队列：连续双周2次触发入池
  - 英文队列：单次即触发入池
- **包含**：
  1. 中途掉线未回拨
  2. 无声电话未回拨
  3. 应回电未回电
  4. 电话回拨错误
  5. 未按静音键（与同事沟通工作内容）
  6. 已知问题未使用方案
  7. 设备判定错误
  8. TIT标记错误（24小时内未补救）
  9. 课堂工单错误（错误删除绑定）

#### 🟢 **第五级：普通扣分（常规扣罚）**
- **性质**：一般业务错误，只扣分不入池
- **包含**：所有评分项目的扣分项
  - 服务规范扣分（开场白、等待、态度、技巧等）
  - case处理扣分（流程、解决程度）
  - 工单回填扣分（类目选择、备注描述）

### 2. 按业务领域分类

#### 📞 **服务规范类扣罚（25%权重）**
1. **开场白/结束语**：未说、说错、不完整
2. **客户等待**：无提示离开、留白5秒以上
3. **服务态度**：语调低沉、不耐烦、反问
4. **服务技巧**：未安抚激动客户、未澄清误解
5. **倾听**：抢话、理解错误、未回应
6. **语言表达**：啰嗦、不清晰、口头语多、未用礼貌用语

#### 💼 **Case处理类扣罚（60%权重）**
1. **处理流程**：未按工作流、错误升级、非最优流程
2. **问题解决程度**：
   - 0分：问题未解决关单
   - 32分：给出建议方案（临时方案）
   - 40分：彻底解决问题

#### 📝 **工单回填类扣罚（15%权重）**
1. **天窗类目选择**：必填项错误
2. **备注描述**：记录不清楚、信息缺失

### 3. 特殊规则体系

#### 🔄 **重大失误累计规则**
1. **一次重大失误**：质检成绩当次为0
2. **当月累积3次重大失误**：质检成绩当月为0
3. **当月累计5次重大失误**：绩效成绩当月为0

#### 🛡️ **满分特殊保护规则**
以下情况仍可得满分（40分）：
1. 定位到问题，技术支持侧无法解决，需反馈产研
2. 客户自行恢复（如设备没电）
3. 客户联系不上，对接相关维护人
4. 联系用户，但用户无法配合
5. 设备不支持，给出方案对方知悉

### 4. 与出入池规则的精确映射

#### ✅ **明确触发入池的扣罚**
1. **"当月为零"**：单次即触发入池（中文队列）
2. **"当通为零"**：
   - 中文队列：连续双周2次触发入池
   - 英文队列：单次即触发入池

#### ❌ **不直接触发入池但影响分数的扣罚**
1. 所有普通扣分项
2. 只影响质检分数，不直接触发入池

#### ⚠️ **最严重后果的扣罚**
1. 团队红线：直接开除
2. 虚假工单：可能劝退
3. 当月为零：严重业务错误

## 🔄 与时间概念的整合

### 1. 时间字段标准化
| 时间概念 | 数据字段名 | 格式 | 示例 |
|----------|------------|------|------|
| 工单产生日 | work_order_date | YYYY-MM-DD | 2026-03-28 |
| 工单抽检日 | check_date | YYYY-MM-DD | 2026-03-29 |
| 质检日报文件发出日 | report_date | YYYY-MM-DD | 2026-03-30 |
| 自然日 | natural_day | YYYY-MM-DD | 2026-03-30 |

### 2. 时间计算规则
```python
# 基于数据字典的时间计算
def calculate_time_chain(filename, data_dict):
    """
    根据日报文件名和数据字典计算完整时间链
    """
    # 从文件名提取工单产生日（如"0328"→"2026-03-28"）
    work_order_date = extract_date_from_filename(filename)
    
    # 根据数据字典中的时间规则计算其他日期
    check_date = work_order_date + timedelta(days=1)  # 工单抽检日
    report_date = check_date + timedelta(days=1)      # 日报发出日
    
    return {
        "work_order_date": work_order_date,
        "check_date": check_date,
        "report_date": report_date,
        "natural_day": report_date  # 自然日=日报发出日
    }
```

### 3. 异常时间处理
- **节假日延迟**：report_date顺延，work_order_date不变
- **合并日报**：多个work_order_date对应一个report_date
- **数据缺失**：使用默认规则或人工确认
